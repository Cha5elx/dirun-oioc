# 业务场景与系统对接说明

## 一、系统角色定位

### 1.1 本项目的定位

```
本项目是一个「中间件服务」，不是独立的业务系统，没有用户登录入口。

┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│    有赞商城  ←─────→  本项目（中间件）  ←─────→  一物一码系统    │
│   (销售渠道)           (数据同步)           (溯源管理)          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

用户登录入口：
├── 有赞商家后台：https://www.youzan.com （管理商品、订单、库存）
└── 一物一码系统：第三方提供的后台地址 （管理溯源码、出入库）
```

### 1.2 本项目的功能

| 功能 | 说明 |
|------|------|
| **无用户界面** | 这是一个纯后端服务，只提供 API 接口 |
| **被动触发** | 通过 Webhook 接收事件，自动执行同步逻辑 |
| **数据桥梁** | 连接有赞和一物一码系统，实现数据互通 |

## 二、核心业务场景

### 场景1：商品采购入库

```
业务背景：
├── 品牌方生产商品，需要入库到仓库
├── 仓库扫码录入防伪码到一物一码系统
└── 需要同步库存到有赞商城，以便销售

流程：
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 仓库扫码  │ ──→ │ 一物一码  │ ──→ │   本项目  │
│ 入库操作  │     │ 推送数据  │     │ 接收Webhook│
└──────────┘     └──────────┘     └──────────┘
                                        │
                                        ▼
                                  ┌──────────┐
                                  │ 有赞商城  │
                                  │ 增加库存  │
                                  └──────────┘

触发方式：一物一码系统调用 POST /webhook/oioc
数据流向：一物一码 → 有赞
```

**业务价值**：
- 入库后自动同步库存，无需人工操作
- 确保线上线下库存一致

---

### 场景2：销售发货（核心场景）

```
业务背景：
├── 消费者在有赞商城下单购买商品
├── 订单需要推送到一物一码系统创建出库单
├── 仓库扫码发货，绑定防伪码
└── 发货信息同步回有赞，完成订单

流程：
步骤1: 订单创建
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 消费者下单 │ ──→ │ 有赞商城  │ ──→ │   本项目  │
│          │     │ 推送订单  │     │ 接收Webhook│
└──────────┘     └──────────┘     └──────────┘
                                        │
                                        ▼
                                  ┌──────────┐
                                  │ 一物一码  │
                                  │ 创建出库单│
                                  └──────────┘

步骤2: 仓库发货
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 仓库扫码  │ ──→ │ 一物一码  │ ──→ │   本项目  │
│ 绑定防伪码 │     │ 推送发货  │     │ 接收Webhook│
└──────────┘     └──────────┘     └──────────┘
                                        │
                                        ▼
                                  ┌──────────┐
                                  │ 有赞商城  │
                                  │ 订单发货  │
                                  │ 写入防伪码│
                                  └──────────┘

触发方式：
├── 步骤1: 有赞调用 POST /webhook/youzan
└── 步骤2: 一物一码调用 POST /webhook/oioc
数据流向：有赞 → 一物一码 → 有赞
```

**业务价值**：
- **防伪溯源**：每个订单绑定唯一的防伪码
- **渠道追溯**：可追溯商品从生产到消费者的完整链路
- **自动发货**：仓库扫码后自动更新有赞订单状态
- **消费者查询**：消费者可通过防伪码查询商品真伪和来源

                                        ▼
                                  ┌──────────┐

                                  │ 创建退货单│
                                  └──────────┘

步骤2: 退货完成
┌──────────┐     ┌──────────┐     ┌──────────┐
│ 仓库扫码  │ ──→ │ 一物一码  │ ──→ │   本项目  │
│ 退货入库  │     │ 推送完成  │     │ 接收Webhook│

                                        │

## 三、溯源码与渠道责任

### 3.1 什么是溯源码（防伪码）

```
溯源码 = 商品的「身份证」

每个商品都有唯一的防伪码，记录：
├── 生产信息：生产日期、批次、生产线
├── 入库信息：入库时间、仓库、数量
├── 出库信息：出库时间、订单号、收货人
├── 物流信息：物流单号、配送路径
└── 消费者：购买时间、购买渠道
```

### 3.2 渠道责任主体追溯

```
查询场景：

消费者扫码查询
      │
      ▼
┌─────────────────────────────────────┐
│         一物一码系统                 │
│                                     │
│  防伪码: 1234567890123              │
│  ├── 产品名称: XXX品牌保健品         │
│  ├── 生产日期: 2026-03-01           │
│  ├── 入库仓库: 华东仓               │
│  ├── 出库时间: 2026-03-15           │
│  ├── 销售渠道: 有赞商城             │
│  ├── 订单号: YZ20260315001          │
│  └── 收货人: 张** (138****1234)     │
│                                     │
│  ✅ 正品验证通过                    │
│                                     │
└─────────────────────────────────────┘

渠道责任追溯：
├── 生产方：品牌方（产品质量责任）
├── 仓储方：仓库运营方（仓储管理责任）
├── 销售方：有赞商家（销售合规责任）
└── 物流方：物流公司（配送责任）
```

### 3.3 业务价值

| 角色 | 价值 |
|------|------|
| **品牌方** | 防伪防窜货，追溯产品流向，管控渠道 |
| **消费者** | 扫码验真伪，查询产品来源，放心购买 |
| **监管部门** | 追溯问题产品，定位责任主体 |
| **商家** | 提升信任度，减少假货投诉 |

## 六、常见问题

### Q1: 部署后需要登录本项目吗？

**不需要。** 本项目是纯后端服务，没有用户界面。部署后只需要：
1. 配置好环境变量（有赞和一物一码的凭证）
2. 配置好 Webhook 回调地址
3. 服务会自动运行

### Q2: 如何验证服务是否正常？

```bash
# 健康检查
curl https://你的域名/health

# 返回
{"status":"ok","timestamp":"2026-04-08T..."}
```

### Q3: 数据存储在哪里？

```
├── 有赞：存储商品、订单、库存数据
├── 一物一码：存储防伪码、出入库记录、溯源数据
└── 本项目：可选存储同步日志（目前记录在日志文件中）
```

### Q4: 如何查看同步日志？

```bash
# 查看日志文件
cat logs/sync.log

# 或在云函数控制台查看日志
```
