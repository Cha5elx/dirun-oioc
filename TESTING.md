## 🧪 多次测试

### 重新运行测试
你可以多次运行测试脚本：
```bash
node tests/flow/simulate-flow.js
```

每次运行都会生成新的数据，前端会实时显示。

### 清空数据
如果需要清空数据，删除数据库文件：
```bash
# Windows
del database\db.json

# Linux/Mac
rm database/db.json
```

然后重启后端服务，会自动创建新的数据库。

## 🐛 常见问题

### 问题1：后端启动失败

**症状**：
```
Error: listen EADDRINUSE: address already in use :::3000
```

**解决方法**：
```bash
# 查找占用3000端口的进程
netstat -ano | findstr :3000

# 结束进程（替换PID为实际进程ID）
taskkill /PID <PID> /F

# 或修改端口
# 编辑 .env 文件，修改 PORT=3001
```

---

### 问题2：前端启动失败

**症状**：
```
Port 5173 is already in use
```

**解决方法**：
```bash
# 查找占用5173端口的进程
netstat -ano | findstr :5173

# 结束进程
taskkill /PID <PID> /F
```

---

### 问题3：测试脚本连接失败

**症状**：
```
❌ 入库请求失败: connect ECONNREFUSED 127.0.0.1:3000
```

**解决方法**：
1. 确认后端服务已启动
2. 确认端口3000没有被占用
3. 检查防火墙设置

---

### 问题4：登录失败

**症状**：输入正确的账号密码仍然登录失败

**解决方法**：
```bash
# 删除数据库文件
del database\db.json

# 重启后端服务
node index.js
```

---

### 问题5：时间显示不正确

**症状**：时间显示为ISO格式或其他时区

**解决方法**：
确认代码已更新：
- `src/services/sync.js` - 使用北京时间
- `src/models/syncLog.js` - 使用北京时间
